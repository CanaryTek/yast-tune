IDE DMA mode

Some IDE devices are broken and they do not work properly if they have DMA mode enabled (especially CD-ROM drives). DMA mode is enabled by kernel after boot, but it can be changed by init script. This YaST2 module is frontend for hdparm utility. There is init script which sets DMA mode after boot.

Module description

In this YaST2 module DMA mode of selected device can be enabled or disabled.

Features of YaST2 module are available here.

Settings are stored in file /etc/sysconfig/hardware. There are variables DEVICES_FORCE_IDE_DMA_ON and DEVICES_FORCE_IDE_DMA_OFF with list of devices (e.g. "hdc hdd") on which DMA will be enabled/disabled. Other devices have their DMA mode unchanged.

Workflow

IDE DMA configuration module has only one dialog.

There are displayed all attached IDE devices in the table. Column DMA setup contains DMA value which will be set after 'Finish' button is pressed and after boot by init script (on, off, no change). Next columns contain device information, in the last column is current DMA setup.

To change DMA mode status select device in the table and push a button below the table with required DMA status.
